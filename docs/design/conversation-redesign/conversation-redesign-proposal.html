01 핵심 결론

5개 서브에이전트와 메인세션이 독립 분석했고 진단·방향이 완전히 일치했습니다.

  • 병목은 데이터가 아니라 표현 레이어다. Step/Turn은 rich 렌더에 필요한 신호(사고·도구 입출력·모델·비용·서브에이전트)를 이미 전부 보유.
  • 지금 가장 큰 결함은 "데이터 결선 버그". Turn 선택 시 응답이 비어 "(no response available)" — 스크린샷의 빈 화면.
  • 해법 = "Turn 대화 리더"로 전환. 순수 헬퍼 ConversationStoryBuilder가 Turn을 큐레이션된 블록으로 변환 → NSStackView 카드 스택으로 렌더.
  • 확장성을 1급 설계 가치로. 표시 대상(테이블·diff·도구묶음·차트…)이 늘어도 "타입 1개 + 렌더러 등록"으로 추가 — 렌더러 레지스트리 + 2단계 파이프라인 + 안전 폴백.
  • 한눈에 훑기 우선. 도구는 "읽기 파일 3개 ›"처럼 묶어 접고, 파일 편집은 diff 카드로, 본문은 마크다운(테이블 포함)으로.

06 확장성 아키텍처 최우선

표시 대상이 계속 늘어난다 — 새 대상 추가 = 작은 타입 1개 + 렌더러 등록. 그 외 코드는 불변이어야 한다.

2단계 렌더 파이프라인

Level 1 — 대화 구조 (누가/무엇을 했나)
Turn(steps[]) ──► [BlockExtractor] Step 해석 규칙 ──► [ConversationBlock] 큐레이션된 블록
                             ▼ 각 블록을 뷰로 (레지스트리)
Level 2 — 콘텐츠 리치 (텍스트 안의 표/코드/리스트)
text.markdown ──► [MarkdownNode] paragraph/heading/list/table/code/quote ──► [NodeRenderer]

두 층을 분리하는 것이 확장성의 핵심. Level 1은 대화 뼈대(화자·도구·사고)와 큐레이션, Level 2는 본문 내부 리치 콘텐츠(테이블이 여기서 NSGridView로 렌더).

확장 포인트 3개 — 모두 "등록만"

① BlockExtractor

Step을 보고 블록 생성. 새 대화 패턴이 생기면 추출기 추가.
예: ToolGroup(묶음), Diff(파일편집), SubAgent

② BlockRenderer

블록 → NSView. 타입별 렌더러를 레지스트리에 등록. 미등록은 평문 폴백.

③ NodeRenderer

마크다운 노드별 리치 렌더.
테이블=NSGridView, 코드=카드, 리스트…

블록 모델 — 하이브리드 (열린 protocol + 핵심 enum)

방식장점단점 / 결론
enum + caseexhaustive switch로 누락 컴파일 에러새 case마다 모든 switch 수정 → 확장 비용↑
protocol + 레지스트리새 타입 = 타입+렌더러 등록만, 기존 불변, 미등록 폴백런타임 폴백으로 안전성 보완 → 채택
안전한 확장

어떤 블록/노드도 렌더러가 없으면 평문으로라도 표시(폴백 불변식). 실험적 표시 대상을 늦게 붙여도 빈 화면·크래시 없음. Tier·Role 같은 핵심 분기만 enum으로 컴파일 안전성 확보.

검증: "5분 추가" 시나리오

테이블 리치하게?

Level 2에 TableNodeRenderer(NSGridView) 등록. 다른 코드 0줄 수정.

mermaid 다이어그램 추가?

MermaidBlock + MermaidExtractor + MermaidRenderer 등록. 기존 빌더/렌더러 불변. 렌더러 붙기 전엔 코드블록으로 폴백.

전문: extensibility-architecture.md (프로토콜 스케치 · 파일 배치 · 성능 전략 포함)

09 큐레이션 — 3-Tier 점진 공개

"한눈에 훑기" = 기본은 압축(스캔 라인), 펼침은 on-demand.

Tier대상표시 방식
Tier 1 · 항상 펼침내 프롬프트 · 모델 최종 답변 · 오류/중단/압축강조 카드 + 마크다운(테이블 포함) + 첨부 칩 / 사유 배너
Tier 2 · 기본 접힘도구 묶음 · 사고 · 서브에이전트 · 파일 diff · 문서 카드아이콘 + 한 줄 요약 → 펼치면 상세
Tier 3 · 숨김/토글전체 도구 입출력 JSON · 시스템 메타 · compact 원문헤더 토글 또는 Raw 탭 escape
통제감 · 배치

헤더에 전역 토글 바(도구 · thinking · 시스템) + Compact↔Full(영속). 기본 Compact로 전체 흐름을 5초에 훑게.

주요 리스크와 완화

  • 카드 스택 전환 복잡도. → 동일선택 re-render 스킵(:537-545) 유지 + 스크롤 앵커 보존.
  • 전체 선택·복사 상실. → 본문 selectable NSTextView + 헤더 "Copy as Markdown".
  • 긴 Turn 성능. → 접힌 블록 본문 lazy 생성 + Show more, 필요 시 가상화.
  • 마크다운 오판(로그/JSON). → 도구 출력은 마크다운 미적용, 답변 본문만.
  • 확장이 성능/안정성 깨지 않게. → 폴백 불변식(렌더러 없어도 평문), 뷰 풀링.
